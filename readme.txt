这个爬虫用于爬取fooooot的轨迹数据，总共的数据体量大概在240w左右条轨迹。
设计的思想如下：
	采用多进程，多线程爬取。存入到文件系统中。
	alggrithm:
		输入爬取用户的id范围，爬取的id与相应的文件比较
		如果记录的文件表明，数据已经爬取完成，那么该文件不用再爬取了。
		用一个文件记录下来。不需要数据库
		
		使用模块1获取数据。
		如果获取完成了数据，则将文件名赋值为1_running->1
				
		
		
		封装的模块1：
		对于每个用户id例如1：
		获取该页面的详细信息HTML：
		http://www.foooooot.com/userindex/1/trip/ 里面有轨迹信息
		（1）获取当前页中的轨迹详情页连接。里面有轨迹的详细数据
			然后依照http://www.foooooot.com/trip/200000/offsettrackjson/  轨迹数据
			http://www.foooooot.com/trip/272737/  里面有轨迹的属性数据
		（2）获取下一页的信息，重复（1）的工作，知道下一页没有为止
		（3）将获取的数据有结构层次的放入到文件中。知道了用户id，轨迹名称，轨迹的内容，轨迹的属性
